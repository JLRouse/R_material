I will use previous session examples to show how to present these in Rmarkdown Rmarkdown can be found here https://rmarkdown.rstudio.com/, with cheatsheet here https://www.rstudio.com/wp-content/uploads/2015/02/rmarkdown-cheatsheet.pdf Rmarkdown provides a way to effectively communicate your code to stakeholders in a reproducible manner, you can choose how much code you want people to see. Have a play around with this Rmarkdown document, make sure all code is pointing at the correct data then add/subtract various chunk code to see how everything changes.

To start an Rmarkdown document of your own open an R script then save it with the .Rmd extension, like this script To output a html document, notice the “output: html_document” code above. To output,press on knit at the top of this screen , and then knit to HTML. You should be taken to another screen and see something similar to the output provided as a HTML document in the learning materials. To run individual code chunks, click on the green arrow in the top right hand corner of the grey boxes. You can use this to test you code chunks before knitting.

You can use “```” to start a code chunk, then control what the viewer sees. You have to start each code chuck with the code type you start with, either r, python etc.
